# Incident Knowledge Base

Цей каталог містить шаблони та журнали, що допомагають стандартизувати реакцію на інциденти, формувати постмортеми та відстежувати виконання коригуючих дій.

## Структура
- `incident_report_template.md` — оперативний шаблон для фіксації деталей під час активного інциденту.
- `postmortem_template.md` — детальний постмортем, який завершується протягом 48 годин після інциденту.
- `action_item_register.md` — централізований реєстр CAPA-елементів.
- `<year>/<incident-id>/` — каталог для фактичних звітів і артефактів (створюється за потреби).

## Процес
1. **Декларація інциденту.** Створіть запис за шаблоном інцидент-репорту, щоб зібрати ключову інформацію для команди.
2. **Мітигація.** Оновлюйте секції про вплив, технічні деталі й коригуючі дії, поки інцидент не стабілізовано.
3. **Постмортем.** Призначте фасилітатора, проведіть аналіз кореневих причин, заповніть шаблон постмортему.
4. **CAPA.** Перенесіть усі дії в `action_item_register.md`, відстежуйте статуси до завершення.
5. **База знань.** Додайте посилання на постмортеми у відповідні runbooks, документацію та тренінги.

## Керування версіями
- Усі зміни шаблонів проходять через pull request, щоб забезпечити узгодженість.
- Для закритих інцидентів створюйте підкаталоги `reports/incidents/<year>/<incident-id>/` із знімками дашбордів, логів та остаточними документами.

Дотримання цього процесу забезпечує прозорість, підзвітність і накопичення знань для зменшення ризику повторних збоїв.
